Users Endpoints

1: Get All Users

Use this endpoint to retrieve a list of all users.

URL: http://localhost:3000/api/users/getPaged

Method: GET

Test this endpoint using Postman by sending a GET request to the above URL.

2: Get User by ID

Use this endpoint to retrieve a user by their ID.

URL: http://localhost:3000/api/users/get/{id}

Method: GET

Replace {id} with the user ID in Postman and send the GET request.

3: Create a New User

Use this endpoint to create a new user by sending data in the request body.

URL: http://localhost:3000/api/users/create

Method: POST

Use Postman to send a POST request with the user data in the body.

Request Body Example:

{
    "firstName": "John",
    "lastName": "Doe",
    "email": "john.doe@example.com"
}

4: Update User by ID

Use this endpoint to update a user's information by their ID.

URL: http://localhost:3000/api/users/update/{id}

Method: PUT

Replace {id} with the user ID and provide updated data in the request body.

Request Body Example:

{
    "firstName": "John",
    "lastName": "Doe Updated",
    "email": "john.doe.updated@example.com"
}

5: Delete User by ID

Use this endpoint to delete a user by their ID.

URL: http://localhost:3000/api/users/delete/{id}

Method: DELETE

Replace {id} with the user ID and send the DELETE request in Postman.

6: Get selected Users

Use this endpoint to retrieve a list of all users with pagination options using limit and offset.

URL: http://localhost:3000/api/users/getPaged?limit=10&offset=0

Method: GET

Test this endpoint using Postman by sending a GET request to the above URL with the query parameters limit and offset:

Request Example:

GET http://localhost:3000/api/users/getPaged?limit=10&offset=0

Explanation: The limit parameter specifies how many users to return, and the offset parameter specifies where to start retrieving the users (i.e., which record to start from).

7: Search Users by Field

Use this endpoint to search users based on a specific field value.

URL: http://localhost:3000/api/users/search

Method: GET

Query Parameters:

  • field: The field to search (e.g., firstName).
  • value: The value to match in the specified field.

Example:

Example Request:

            http://localhost:3000/api/users/search?field=firstName&value=John
